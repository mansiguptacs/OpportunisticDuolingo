We rebuilt our analytics path as a streaming lakehouse

After years of batch ETL, our team moved to a streaming lakehouse. The hardest lessons were not the fancy sinks — they were partitioning, watermarks, and compaction discipline.

Partitioning: A low-cardinality customer_tier key created one hot partition while others sat idle. We switched to a hashed entity id aligned with query patterns and lag collapsed overnight.

Watermarks: Event-time windows never closed until we set an explicit watermark policy for late data. Late arrivals go to a side output instead of blocking the main pipeline.

Exactly-once & compaction: Idempotent sinks plus periodic small-file compaction kept object-storage scans cheap. Without compaction, query planners drowned in tiny parquet files.

If you are stuck between “batch forever” and “pure stream everything,” start with watermark correctness and partition skew before chasing new engines.
